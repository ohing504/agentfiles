## 2026-03

### Boris Tane — How I Use Claude Code

- **URL:** https://boristane.com/blog/how-i-use-claude-code/
- **요약:** 9개월간 Claude Code를 주력 개발 도구로 사용한 경험. research.md → plan.md → annotation cycle → execute의 3단계 워크플로우. 마크다운 파일을 "공유 가변 상태"로 활용하며, 사람이 항상 운전석에 있어야 한다고 강조.
- **인사이트:** 결국 개발자 각자가 본인만의 워크플로우를 설계하고 다듬어야 한다는 점이 인상적이었다. oh-my-opencode 같은 올인원 도구는 비개발자에게 매력적이지만, 실전에서는 블랙박스가 너무 많다. 본인이 이해하고 통제할 수 있는 워크플로우가 결국 강하다.

### 박진형 — Claude Code 세계 1위 개발자의 AI 워크플로우

- **URL:** https://yozm.wishket.com/magazine/detail/3630/
- **요약:** "토큰을 많이 쓴다고 좋은 건 아니다." 빠른 피드백 루프가 핵심. 도메인 전문성이 AI 활용 효율을 결정하며, 프레임워크보다 본인만의 규칙 체계가 중요하다고 강조. SDD(Spec-Driven Development)의 부활.
- **인사이트:** 도구나 프레임워크에 의존하기보다, 빠른 피드백 루프를 설계하는 게 핵심이라는 메시지. agentfiles가 단순 설정 관리자가 아니라 "워크플로우를 이해하고 가꾸는 도구"로 진화해야 한다는 방향성을 확인시켜준 글.
