> Foundation — 이해 레이어 + 점진적 UI 확장

설계 결정: 기존 v1 레이아웃을 유지하면서 AI 기능을 점진적으로 추가한다. UI 전면 재설계는 하지 않는다.

## Task 1: AIService 인프라

**목표:** Claude CLI pipe 모드를 활용한 AI 서비스 계층 + 파일해시 기반 캐싱

### 생성할 파일

**`src/services/ai-service.ts`**

```typescript
// Claude CLI pipe 모드로 프롬프트 실행
async function runClaudePipe(prompt: string): Promise<string>
// → spawn("claude", ["--pipe"], { stdin: prompt })
// → claude-cli.ts의 execClaude 패턴 참고하되, stdin을 pipe로 전달

// 파일해시 기반 캐시 키 생성
function getCacheKey(filePath: string, lang: string, type: "summary" | "translation"): string
// → md5(fileContent) + lang + type

// 캐시 읽기/쓰기
async function readCache(cacheKey: string): Promise<string | null>
async function writeCache(cacheKey: string, content: string): Promise<void>
// → 저장 위치: ~/.claude/agentfiles/ai-cache/{summaries,translations}/

// 공개 API
export async function summarize(filePath: string, lang: string): Promise<string>
export async function translate(filePath: string, lang: string): Promise<string>
// → 캐시 확인 → 있으면 반환 → 없으면 CLI 호출 → 캐시 저장 → 반환
```

**`src/server/ai.ts`**

```typescript
export const summarizeFn = createServerFn({ method: "POST" })
  .inputValidator(z.object({ filePath: z.string(), lang: z.string() }))
  .handler(async ({ data }) => {
    const { summarize } = await import("@/services/ai-service")
    return summarize(data.filePath, data.lang)
  })

export const translateFn = createServerFn({ method: "POST" })
  .inputValidator(z.object({ filePath: z.string(), lang: z.string() }))
  .handler(async ({ data }) => {
    const { translate } = await import("@/services/ai-service")
    return translate(data.filePath, data.lang)
  })
```

### 수정할 파일

- `src/services/claude-cli.ts` — `execClaudePipe(prompt)` 헬퍼 추가 (stdin pipe 패턴)

### 검증 기준

- [ ] `summarize("/path/to/skill.md", "ko")` → 한국어 요약 반환
- [ ] 동일 파일 재호출 시 캐시에서 즉시 반환 (CLI 호출 없음)
- [ ] 파일 내용 변경 시 캐시 미스 → 새로 생성

```typescript
export function useSummaryQuery(filePath: string, lang: string) {
  return useQuery({
    queryKey: ["ai", "summary", filePath, lang],
    queryFn: () => summarizeFn({ data: { filePath, lang } }),
    staleTime: Infinity,  // 캐시 기반이므로 수동 무효화만
    enabled: !!filePath && !!lang,
  })
}
```

## Task 4: 번역 프리뷰

**목표:** 기존 상세 패널에 Translated 탭 추가, 파일 전문을 선호 언어로 번역

```text
DetailPanel
├── Header (기존)
├── Tabs: [Overview | Source | Translated]
│   ├── Overview: AISummaryCard (Task 3)
│   ├── Source: DetailView (기존)
│   └── Translated: TranslatedView (NEW)
```

**새 컴포넌트:**

`src/components/TranslatedView.tsx`

```typescript

// 내부:

// 3. FileViewer로 번역된 마크다운 렌더링
// 4. 원문/번역 전환 토글 (선택사항)
```

**React Query 훅 (`src/hooks/use-ai.ts` 확장):**

```typescript

  return useQuery({
    queryKey: ["ai", "translation", filePath, lang],

    staleTime: Infinity,
```

### 설계 결정

- 번역은 요약보다 토큰을 많이 사용하므로, **사용자가 탭을 클릭할 때만** 호출 (lazy load)
- 이미 선호 언어가 영어("en")이고 파일도 영어면 탭 숨김

### 검증 기준

- [ ] Translated 탭 클릭 시 번역 로딩 → 결과 표시
- [ ] 번역이 마크다운 렌더링으로 표시
- [ ] 선호 언어가 "en"이면 Translated 탭 숨김
- [ ] 캐시 동작 (동일 파일+언어 재요청 시 즉시 반환)

- 활성 hooks 목록
- MCP 서버 목록

**프롬프트 구조:**

- 페이지 이탈 시 히스토리 초기화 (v2.0에서는 별도 저장 불필요)
